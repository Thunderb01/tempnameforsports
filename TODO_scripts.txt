1. ELIGIBILITY YEARS BACKFILL
   Script: backfill_eligibility.py
────────────────────────────────────────────────────────────────────────────────

Step 1 — Supabase SQL Editor:
  ALTER TABLE public.players ADD COLUMN IF NOT EXISTS eligibility_years integer;

Step 2 — Run script:
  python backfill_eligibility.py --dry-run    (preview only)
  python backfill_eligibility.py              (write to DB)

What it does:
  Reads every player's current `year` value, maps it to a normalized label
  ("Junior" → "Jr") and an eligibility_years int ("Jr" → 2), then writes
  both back. Flags anything it can't recognize so you can fix manually
  in Admin → Players.

2. ARCHETYPES BACKFILL
   Script: backfill_archetypes.py
────────────────────────────────────────────────────────────────────────────────

Step 1 — Supabase SQL Editor:
  ALTER TABLE public.players ADD COLUMN IF NOT EXISTS archetype text;

Step 2 — Run script:
  python backfill_archetypes.py --analyze     (see distribution + borderline players)
  python backfill_archetypes.py --dry-run     (preview changes without writing)
  python backfill_archetypes.py               (write to DB)
  python backfill_archetypes.py --overwrite   (re-compute already-set archetypes)

Step 3 — After confirming archetypes look right, drop old tag columns:
  ALTER TABLE public.players
    DROP COLUMN IF EXISTS playmaker_tags,
    DROP COLUMN IF EXISTS specialist_tags,
    DROP COLUMN IF EXISTS shooting_tags,
    DROP COLUMN IF EXISTS shotmaking_tags,
    DROP COLUMN IF EXISTS interior_tags,
    DROP COLUMN IF EXISTS defensive_tags;

What it does:
  Computes archetype from BTP metrics (SEI, ATH, RIS, DDS, CDI) for every
  player using the same logic as PlayerFinder. Writes result to players.archetype.
  Skips unevaluated players (no metrics) and players with no position set.

4. INTERNATIONAL PLAYER METRICS
   Script: international_metrics.py
────────────────────────────────────────────────────────────────────────────────

Prereqs (run once if not done already):
  - international_players + international_players_stats already exist
  - You've scraped at least one league with liga_u_scraper.py
    (the script reads from those tables — it doesn't scrape)

Run script:
  python international_metrics.py --dry-run                 (preview only)
  python international_metrics.py                           (write to DB)
  python international_metrics.py --league "Spanish Liga U" (one league only)
  python international_metrics.py --season 2026             (one season only)

What it does:
  For every international player, computes 5 metrics — all percentile-ranked
  within their league — and writes them to international_players.metrics:

    offensive_footprint   OFP%   (FGA + 0.44·FTA + TOV + AST)·(TmMP/5)
                                 / (MP · (TmFGA + 0.44·TmFTA + TmTOV + TmAST))
    defensive_score       DDS-style: per-36 stl + 0.75·blk + 0.5·drb - 0.5·pf/36
    winning_impact        Wins-minus-losses performance differential  ← needs splits
    sos_performance       Above-.500 minus below-.500 differential    ← needs splits
    translation_grade     (CQI·.30 + WID·.25 + DEF·.22 + OFP·.23)
                          × tier_mult (T1=1.00, T2=0.90, T3=0.80, T4=0.70)
                          + height_boost ((h_cm - pos_avg) × 0.1, capped ±5)
                          Positional averages: Guard 188, Wing 195, Forward 203, Big 205

Limitation:
  winning_impact and sos_performance need per-player win/loss and opponent-
  strength splits. These live in international_players_splits (currently empty
  unless liga_u_scraper.py was run with split scraping enabled). If those rows
  aren't present, the metric is left null and translation_grade is null too.
  TODO: extend the scraper to fetch the Wins/Losses/vs-.500-Above/Below splits.
